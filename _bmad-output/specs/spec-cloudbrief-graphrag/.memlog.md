updated: 2026-07-09T13:42

- (decision) 按知识库选择性启用 GraphRAG，默认关闭，避免一刀切成本
- (decision) GraphRAG 采用生成阶段增强方案：在 GenerationPipeline 中注入图谱上下文，不改造现有 RetrievalPipeline
- (decision) 图数据库选型 Neo4j 5 Community，通过 Docker Compose 与现有服务统一部署
- (decision) 实体/关系抽取采用 LLM + spaCy 混合策略，PoC 以 LLM 为主验证效果
- (decision) 图数据按知识库隔离：所有节点/关系携带 kb_id 属性，Cypher 查询强制参数化过滤
- (decision) 图谱构建复用现有 Celery + Redis Pub/Sub + SSE 事件机制，新增 graph_extraction / graph_building / graph_indexing_complete 阶段
- (decision) 保持模块化单体架构：新增 app/stores/graph_store.py、app/stages/graph_rag_context_stage.py、app/services/graph_extraction.py、app/tasks/graph_indexing.py
- (decision) 试点知识库优先选择实体关系密集型企业资料（组织架构、高管履历、供应链）
- (constraint) 禁止修改现有 RetrievalPipeline 与 GenerationPipeline 的默认行为，GraphRAG 必须可插拔
- (constraint) 所有 Cypher 查询必须参数化，禁止字符串拼接；kb_id 必须作为强制查询参数
- (constraint) GraphRAG 对所有知识库默认关闭，启用需管理员显式配置
- (constraint) 不引入新的微服务，所有 GraphRAG 模块运行在现有 FastAPI + Celery 进程中
- (non_goal) MVP 不支持跨知识库统一大图
- (non_goal) MVP 不支持全自动零监督通用图谱构建
- (non_goal) MVP 不支持复杂多跳 Agent 推理
- (non_goal) MVP 不要求实时图谱更新，分钟/小时级延迟可接受
- (capability) CAP-1: 管理员可按知识库启用/关闭 GraphRAG 并配置实体/关系 schema
- (capability) CAP-2: 系统可从文本 chunks 中抽取实体与关系（LLM + spaCy 混合）
- (capability) CAP-3: 系统可将抽取结果写入 Neo4j 并按 kb_id 隔离持久化
- (capability) CAP-4: 系统可在生成阶段向 LLM prompt 注入图谱上下文且不影响现有检索链路
- (capability) CAP-5: 系统可通过 Celery + Redis Pub/Sub + SSE 推送图谱构建进度
- (capability) CAP-6: 管理后台提供 GraphRAG 开关、schema 配置与构建任务状态展示
- (capability) CAP-7: 当图谱不可用或查询无法映射到图时系统自动回退到向量 RAG
- (assumption) 当前 query 日志中至少 15% 的问题属于跨实体关系型问题（待 Phase 1 验证）
- (assumption) 试点知识库内容质量高且实体关系密集
- (question) 试点知识库具体选择哪一个？
- (question) schema 由管理员手动定义还是由系统自动生成/推荐？
- (question) Phase 2 是否需要 shadow mode 同时运行 GraphRAG 但不影响答案仅记录差异？
- (note) 成功信号：试点知识库关系型问题子图召回率≥75%、准确率≥75%、GraphRAG 开销≤向量 RAG 30%
- (event) 自验证 Pass 1（一致性）：SPEC.md 满足 Spec Law 1-6 与 8；7 个 capability 均含 intent/success；4 条非目标；成功信号可测试
- (event) 自验证 Pass 2（完整性）：关键源主张（按 KB 选择性、生成阶段增强、Neo4j、混合抽取、kb_id 隔离、Celery/SSE、模块化单体、回退、三阶段路线、参数化安全）均已落入 SPEC.md 或 companions
- (decision) schema 由系统自动生成/推荐：基于 chunks 采样 + LLM 推荐实体/关系类型，管理员可在后台一键确认或微调，降低配置门槛
- (decision) Phase 2 启用 shadow mode：生成阶段同时调用 GraphRAG 但不影响最终答案，仅记录与向量 RAG 的差异用于评估
- (event) 开放问题已解决：schema 自动生成/推荐；Phase 2 需要 shadow mode；试点知识库待 Phase 1 启动前最终确认
- (event) 已生成 Story/任务清单：_bmad-output/planning-artifacts/epics-cloudbrief-graphrag.md 与 _bmad-output/specs/spec-cloudbrief-graphrag/implementation-tasks.md
- (decision) GraphRAG 启用决策点前移到知识库创建环节：用户在新建知识库目录时可自主选择是否启用 GraphRAG，系统在旁边给出场景提示
- (decision) 创建时只做开关选择，schema 配置与图谱重建保留在后台完成，避免打断创建流程
- (decision) Phase 1 不再指定唯一试点知识库，改为准备 2–3 个典型场景样本 KB（组织架构、高管履历、供应链）验证技术与自动 schema 推荐效果
- (decision) Phase 3 评估维度从"是否全面推广"改为"按场景决定是否推广"
