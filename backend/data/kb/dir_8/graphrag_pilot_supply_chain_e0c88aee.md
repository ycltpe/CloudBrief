# CloudBrief 供应链与项目协作关系

本文档描述 CloudBrief 公司内部主要产品线、供应商、项目负责人及跨部门协作关系，用于验证 GraphRAG 对关系型问题的回答能力。

## 一、公司组织架构

CloudBrief 下设三大核心部门：

- **产品研发部**：负责所有产品线的技术实现与迭代。部门负责人为 **李雷**，向 CEO **韩梅梅** 汇报。
- **供应链管理部**：负责供应商准入、采购议价与交付风险管理。部门负责人为 **张伟**，向 CEO **韩梅梅** 汇报。
- **客户成功部**：负责大客户交付、售后支持与续约。部门负责人为 **王芳**，向 CEO **韩梅梅** 汇报。

CEO 韩梅梅 直接管理以上三位部门负责人，并兼任公司董事长。

## 二、产品线与项目负责人

公司当前有三条主要产品线：

### 1. CloudBrief 知识库（KB）
- **产品负责人**：李雷（兼任产品研发部负责人）
- **技术负责人**：赵敏
- **当前版本**：v2.1
- **依赖供应商**：
  - 向量数据库由 **MilvusTech** 提供
  - Embedding 服务由 **DashScope** 提供

### 2. CloudBrief 智能客服（CS）
- **产品负责人**：王芳（兼任客户成功部负责人）
- **技术负责人**：刘洋
- **当前版本**：v1.5
- **依赖供应商**：
  - 大模型对话服务由 **DashScope** 提供
  - 语音转文字服务由 **讯飞星火** 提供

### 3. CloudBrief 数据分析平台（DA）
- **产品负责人**：张伟（兼任供应链管理部负责人）
- **技术负责人**：陈静
- **当前版本**：v3.0
- **依赖供应商**：
  - 云存储与计算资源由 **阿里云** 提供
  - BI 可视化组件由 **帆软** 提供

## 四、重点项目

### 项目 Alpha：知识库 GraphRAG 增强
- **项目代号**：Alpha
- **项目经理**：赵敏
- **参与部门**：产品研发部、供应链管理部
- **项目目标**：在 KB 产品中引入图数据库，提升跨文档关系型问答能力
- **依赖供应商**：MilvusTech（向量库）、DashScope（抽取模型）
- **当前状态**：进行中，预计 2026-08 上线

### 项目 Beta：智能客服多模态升级
- **项目代号**：Beta
- **项目经理**：刘洋
- **参与部门**：客户成功部、产品研发部
- **项目目标**：为 CS 产品增加语音输入与富媒体回复能力
- **依赖供应商**：讯飞星火（语音识别）、DashScope（对话生成）
- **当前状态**：需求评审中

### 项目 Gamma：数据平台实时化
- **项目代号**：Gamma
- **项目经理**：陈静
- **参与部门**：供应链管理部、产品研发部
- **项目目标**：将 DA 平台从 T+1 报表升级为实时看板
- **依赖供应商**：阿里云（实时计算）、帆软（可视化）
- **当前状态**：开发中，预计 2026-09 上线

## 五、跨部门协作关系

- 李雷 负责协调 产品研发部 对 Alpha、Beta、Gamma 三个项目的技术支持。
- 张伟 负责协调 供应链管理部 对 Alpha、Gamma 项目的供应商资源。
- 王芳 负责协调 客户成功部 对 Beta 项目的客户需求输入。
- 赵敏 与 陈静 共同参与 Gamma 项目的技术架构评审。
- 刘洋 与 赵敏 共同参与 Beta 项目的多模态方案设计。

## 六、关键关系总结

- 汇报关系：李雷、张伟、王芳 均向 韩梅梅 汇报。
- 产品-技术负责人：KB-赵敏、CS-刘洋、DA-陈静。
- 供应商-产品：DashScope → KB + CS；讯飞星火 → CS；MilvusTech → KB；阿里云 + 帆软 → DA。
- 项目-参与方：Alpha → 赵敏 + 产品研发部 + 供应链管理部；Beta → 刘洋 + 客户成功部 + 产品研发部；Gamma → 陈静 + 供应链管理部 + 产品研发部。
