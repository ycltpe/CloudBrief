# CloudBrief Admin Console — Experience Spine

## Foundation

**Form factor**: 桌面优先的响应式 Web 应用，主要使用场景为 13–27 英寸显示器上的管理操作。

**UI system**: Next.js 14 + React + Tailwind CSS + lucide-react。不使用 shadcn/ui，但遵循类似的组件化、无衬线、高信息密度的 SaaS 工具风格。`DESIGN.md` 是视觉身份参考，定义颜色、字体、圆角、间距和组件 token。

**用户角色**: `admin`（管理员）、`qa`（质检/客服主管）、`user`（普通支持人员）。左侧菜单根据角色动态显示。

## Information Architecture

| Surface | Route | Reached from | Purpose | 最低角色 |
|---|---|---|---|---|
| 登录页 | `/admin/login` | 未认证访问 `/admin/*` | 用户名密码登录 | 公开 |
| 注册页 | `/admin/register` | 登录页链接 | 新用户注册，默认 `user` | 公开 |
| Dashboard | `/admin/dashboard` | 登录成功 / 侧边栏 | 系统概览：用户数、会话数、索引状态、最近评测 | user |
| 系统设置 | `/admin/settings` | 侧边栏 | 修改拒答阈值、时效阈值等运行参数 | admin |
| 用户管理 | `/admin/users` | 侧边栏 | 用户列表、新增、删除 | admin |
| 聊天助手 | `/admin/chat` | 侧边栏 | 内嵌现有 Chat 组件，继续使用 RAG 问答 | user |
| 知识库管理 | `/admin/kb` | 侧边栏 | 目录树、文件上传、删除、重建索引 | admin |
| RAGAS 评测审计 | `/admin/eval` | 侧边栏 | 评测记录列表、详情、人工反馈 | qa |
| 评测详情 | `/admin/eval/[id]` | 评测列表 | 单条评测的完整过程与人工复核 | qa |

**布局结构**：所有 `/admin/*` 路由共享统一布局 `app/admin/layout.tsx`。
- 左侧固定侧边栏（`240px`），顶部为产品 Logo + 名称。
- 右侧为顶部栏（`56px`）+ 滚动内容区。
- 未登录用户访问 `/admin/*` 自动重定向到 `/admin/login`。

## Voice and Tone

管理后台面向内部专业人员，语气**直接、清晰、不煽情**。

| Do | Don't |
|---|---|
| "索引重建完成" | "🎉 太棒了，索引重建成功啦！" |
| "删除后无法恢复，是否继续？" | "确定要删除吗？" |
| "当前知识库中未找到相关信息" | "抱歉，我找不到答案" |
| "来源较旧，请核对最新产品版本" | "警告：该答案可能已过期" |
| "共 12 名用户" | "您当前拥有 12 名用户" |

按钮文案使用动词开头："保存"、"创建用户"、"上传文件"、"重建索引"。

## Component Patterns

### 侧边栏菜单项

- 图标 + 文字，垂直排列。
- 当前路由高亮：左侧 `2px` 主色竖线 + 浅灰背景。
- 悬停状态：浅灰背景。
- 角色无权限的菜单项不显示（而非显示禁用态）。

### 顶部栏

- 左侧：页面标题（如"用户管理"）。
- 右侧：当前用户名 + 角色标签 + 头像下拉菜单（个人资料 / 退出登录）。

### 统计卡片（Dashboard）

- 4 列网格，每卡片展示一个指标。
- 指标值使用 `display` 样式，标签使用 `muted-foreground`。
- 趋势或状态用右侧小徽章表示。

### 数据表格（用户管理、评测列表、文件列表）

- 表头固定样式，行 hover 变浅灰。
- 操作列靠右，使用图标按钮。
- 空状态时显示居中的图标 + 文字 + 主操作按钮。
- 分页器位于表格右下角。

### 表单（设置、新增用户、新建目录）

- 标签左对齐，输入框占满容器。
- 必填项标红星号。
- 提交按钮在表单右下角，次要按钮在左侧。
- 保存成功后显示 Toast。

### 目录树（知识库管理）

- 左侧目录树，右侧文件列表 + 操作面板。
- 目录节点支持展开/收起。
- 右键菜单或操作按钮：新建目录、删除目录（仅空目录）。
- 选中目录后，右侧显示该目录下的文件列表和"上传文件"、"重建索引"按钮。

### 索引重建进度面板

- 复用现有 `IndexRebuildPanel` 组件。
- 位于知识库管理页面顶部或右侧，作为固定操作区。

### 评测审计详情

- 左右分栏：左侧问题/答案/标准答案，右侧 RAGAS 分数和人工评分表单。
- 检索到的片段以折叠卡片展示，标记每个片段是否被 RAGAS 判定 relevant。

## State Patterns

| State | Surface | Treatment |
|---|---|---|
| 未认证访问 | `/admin/*` | 重定向到 `/admin/login` |
| 登录中 | 登录页 | 主按钮显示 loading spinner，禁用输入 |
| 登录失败 | 登录页 | 输入框下方显示红色错误文字 |
| 页面加载 | 任意内容页 | 使用 Skeleton 占位，保持布局结构 |
| 空列表 | 用户管理 / 评测列表 / 文件列表 | 居中图标 + "暂无数据" + 主操作按钮 |
| 删除确认 | 任意删除操作 | 弹窗二次确认，删除按钮为 destructive 样式 |
| 保存成功 | 表单页 | 右上角 Toast："保存成功" |
| 保存失败 | 表单页 | 字段级错误 + 全局错误提示 |
| 索引重建中 | 知识库管理 | 进度面板显示步骤状态，禁用"重建索引"按钮 |
| 索引重建完成 | 知识库管理 | Toast + 进度面板显示"全部完成" |
| 权限不足 | 受保护页面 | 隐藏入口；直接访问 URL 返回 403 页面 |
| 网络错误 | 任意页面 | Toast："网络异常，请稍后重试" |

## Interaction Primitives

### 导航

- 侧边栏点击切换路由，当前项高亮。
- 顶部栏显示当前页面标题。
- 面包屑非必须，复杂详情页可用（如 `RAGAS 审计 / 详情 #123`）。

### 表单

- `Enter` 在单字段表单中提交（如登录）。
- `Esc` 关闭对话框。
- 表单字段失焦时即时校验（可选），提交时全量校验。

### 表格

- 点击行可进入详情（如用户详情、评测详情）。
- 悬停行显示操作按钮。
- 分页器支持上一页/下一页和页码跳转。

### 文件上传

- 支持拖拽上传和点击选择文件。
- 上传中显示进度条或 spinner。
- 上传成功后刷新文件列表。

### 索引重建

- 点击"重建索引"按钮后禁用按钮，显示进度。
- 支持 SSE 实时更新或轮询任务状态。
- 失败时显示错误详情和"重试"按钮。

## Accessibility Floor

- 目标 WCAG 2.2 AA。
- 所有交互元素可通过键盘到达和操作。
- 表单字段与标签正确关联（`htmlFor`）。
- 错误信息使用 `aria-live` 播报。
- 图标按钮使用 `aria-label` 说明操作。
- 模态对话框焦点锁定，关闭后焦点返回触发元素。
- 颜色不是唯一的状态标识（例如状态徽章同时用文字和颜色）。

## Responsive & Platform

| Breakpoint | Behavior |
|---|---|
| `≥ 1024px` | 侧边栏固定展开，内容区宽表格布局。 |
| `768–1023px` | 侧边栏收起为图标栏，点击展开。 |
| `< 768px` | 侧边栏隐藏，顶部栏显示汉堡菜单，点击后从侧面滑出。内容区单列堆叠。 |

管理后台主要使用场景为桌面端，移动端以可访问和应急查看为主。

## Inspiration & Anti-patterns

### 借鉴

- **Linear**: 干净的信息密度、明确的按钮层级、简洁的 Toast 反馈。
- **Vercel Dashboard**: 侧边栏 + 顶部栏 + 卡片式指标布局。
- **Supabase**: 表格空状态设计、表单字段分组。

### 避免

- 不要使用复杂的仪表盘图表（如 D3 复杂可视化），Dashboard 以数字卡片和简单列表为主。
- 不要给每个操作都加确认弹窗，只保留删除等不可逆操作。
- 不要使用消费级产品的情绪化文案和动画。
- 不要让不同角色看到相同的侧边栏再提示无权限，应直接隐藏无权限入口。

## Key Flows

### Flow 1 — 管理员为新团队开通账号并上传知识库

**主角**: 老王，CloudBrief 系统管理员。

1. 老王打开 `/admin/login`，输入用户名密码登录。
2. 进入 Dashboard，看到当前有 3 名用户、今日 12 次会话、索引状态正常。
3. 点击左侧"用户管理"，进入用户列表页。
4. 点击"新增用户"按钮，填写用户名 `linxiaolin`、初始密码、角色 `user`，保存。
5. **小高潮**：新用户出现在列表中，Toast 提示"用户创建成功"。
6. 老王点击左侧"知识库管理"，进入目录树页面。
7. 点击"新建目录"，输入"产品更新 Q3"，创建成功。
8. 选中该目录，点击"上传文件"，选择本季度的 `changelog_q3.json` 和 `faq_q3.md`。
9. 文件上传成功后出现在右侧文件列表。
10. 老王点击"整合重建索引"，进度面板显示"解析 → 切分 → Embedding → 写入向量库 → 重建 BM25 → 原子切换索引"各步骤状态。
11. **高潮**：索引重建完成，进度面板显示"全部完成"，Toast 提示"索引已更新，新内容可检索"。
12. 老王切换到"聊天助手"，输入"Q3 更新了哪些功能"，验证新内容可被检索到。

**失败分支**：
- 上传文件格式不支持 → 文件列表旁显示红色错误："仅支持 Markdown/JSON/CSV 文件"。
- 重建索引失败 → 进度面板失败步骤高亮，显示错误日志和"重试"按钮。

---

### Flow 2 — 客服主管审计一次低分评测

**主角**: 张姐，客服主管（角色 `qa`）。

1. 张姐登录后进入 Dashboard，看到最近评测平均分卡片中 faithfulness 偏低。
2. 点击左侧"RAGAS 评测审计"，进入评测列表页。
3. 列表按 faithfulness 分数升序排列，她发现一条分数为 0.65 的记录。
4. 点击"查看详情"，进入 `/admin/eval/42`。
5. 详情页左侧显示问题、系统答案、标准答案；右侧显示 RAGAS 五维分数。
6. 她展开"检索到的片段"，看到系统引用了 3 个片段，其中只有 1 个被 RAGAS 判定为 relevant。
7. 她认为答案虽然引用了正确片段，但遗漏了关键步骤 4。
8. 在右侧人工评分区给出 3 分，备注"需补充步骤 4"，勾选"需修改"，点击"保存反馈"。
9. **高潮**：页面显示"反馈已保存"，列表页该记录的人工评分列更新为 3 分，她可以导出这批标注给开发团队。

**失败分支**：
- 提交反馈时网络中断 → Toast 提示"保存失败，请重试"，表单保留已填写内容。

---

### Flow 3 — 普通支持人员从后台进入聊天

**主角**: 小林，新入职客服（角色 `user`）。

1. 小林登录后进入 Dashboard，看到今日已有若干会话。
2. 她点击左侧"聊天助手"，右侧加载聊天界面。
3. 界面与现有首页 Chat 组件一致，她输入"报表导出后无法打开怎么办"。
4. 副驾返回答案和引用，她点击引用查看原文摘要。
5. **高潮**：她确认答案无误后复制给客户，无需切换页面。

**失败分支**：
- 答案触发拒答 → 界面显示拒答文案"根据当前知识库，我找不到足够信息回答这个问题"，不显示引用。

## Open Questions / Notes

1. Dashboard 的"今日会话数"需要 `conversations` 表增加 `user_id` 字段。
2. 是否需要用户详情页？当前 PRD 只要求列表/新增/删除，详情页可作为增强。
3. 知识库文件上传后是否显示文件内容预览？MVP 建议只显示元数据，不预览。
4. RAGAS 评测审计列表默认排序：按创建时间倒序，还是按最低分数排序？建议默认按时间倒序，提供按分数排序。
